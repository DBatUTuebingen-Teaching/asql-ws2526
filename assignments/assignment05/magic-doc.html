Magic the Gathering Card Deck by MTGJSON

Background

Magic the Gathering is a collectible card game. Each player has their own deck of cards. In game play, they build their deck out of at least 60 cards. New cards are released on a regular basis through expansion sets. All cards belong either to the base sets or some expansion or compilation set.

magic-cards.json

The magic-cards.json file is organized by card name and merges card data from multiple sets into a single card object.

Due to this merging of card data, several set specific fields are not present in the cards data, e.g., artist, border, flavor, foreignNames, id, mciNumber, multiverseid, number, originalText, originalType, rarity, releaseDate, reserved, timeshifted, variations, watermark etc.

This file is lighter weight and is meant to provide a quick and easy way to get basic card data. (In case you don't find your favourite Magic card, sorry, this data set is not very up to date.)

The JSON file itself looks like this:

{
...
      "Nevinyrral's Disk" : { /* card data */ },
                 "Norrit" : { /* card data */ },
"Niv-Mizzet, Dracogenius" : { /* card data */ },
...
}

Cards

The card data is a JSON object with key/value pairs. Below you will find a table detailing each key.

Key Example Description
layout "normal" The card layout. Possible values: normal, split, flip, double-faced, token, plane, scheme, phenomenon, leveler, vanguard, meld
name "Niv-Mizzet" The card name. For split, double-faced and flip cards, just the name of one side of the card. Basically each 'sub-card' has its own record.
manaCost "{2}{U}{U}{R}{R}" The mana cost of this card. Consists of one or more mana symbols.
cmc 6 Converted mana cost. Always a number. NOTE: cmc may have a decimal point as cards from unhinged may contain "half mana" (such as 'Little Girl' with a cmc of 0.5). Cards without this field have an implied cmc of zero as per rule 202.3a
colors [ "Blue", "Red" ] The card colors. Usually this is derived from the casting cost, but some cards are special (like the back of double-faced cards and Ghostfire).
type "Legendary Creature - Dragon Wizard" The card type. This is the type you would see on the card if printed today.
supertypes [ "Legendary" ] The supertypes of the card. These appear to the far left of the card type. Example values: Basic, Legendary, Snow, World, Ongoing
types [ "Creature" ] The types of the card. These appear to the left of the dash in a card type. Example values: Instant, Sorcery, Artifact, Creature, Enchantment, Land, Planeswalker
subtypes [ "Dragon", "Wizard" ] The subtypes of the card. These appear to the right of the dash in a card type. Usually each word is its own subtype. Example values: Trap, Arcane, Equipment, Aura, Human, Rat, Squirrel, etc.
text "Flying Whenever Niv-Mizzet, Dracogenius deals damage to a player, you may draw a card. {U}{R}: Niv-Mizzet, Dracogenius deals 1 damage to target creature or player." The text of the card. May contain mana symbols and other symbols.
power "5" The power of the card. This is only present for creatures. This is a string, not an integer, because some cards have powers like: "1+*"
toughness "5" The toughness of the card. This is only present for creatures. This is a string, not an integer, because some cards have toughness like: "1+*"
imageName "niv-mizzet, dracogenius" This used to refer to the mtgimage.com file name for this card.
This field will continue to be set correctly and is now only useful for UID purposes.
colorIdentity [ "U", "G" ] This is created reading all card color information and costs. It is the same for double-sided cards (if they have different colors, the identity will have both colors). It also identifies all mana symbols in the card (cost and text). Mostly used on commander decks.
